19//! Utility library for writing end-to-end tests for Gear programs.
20//!
21//! This crate can be considered a companion of the
22//! [`gtest`](https://docs.gear.rs/gtest/) when covering the program code
23//! with tests. When `gtest` is most appropriate for unit and integration
24//! tests, `gclient` fits better for higher-level debugging.
25//!
26//! `gclient` is intended to test Gear programs with a real blockchain network.
27//! It allows you to send extrinsics and RPCs by connecting to the network.
28//!
29//! It is essential to underline that testing with `gclient` requires the
30//! running node as the second part of the test suite. The `gclient` interacts
31//! with the node over the WebSocket protocol. Depending on the purpose of
32//! testing, `gclient` can communicate with either a local or a remote node. The
33//! best choice is to use the **local node in developer mode** for initial
34//! debugging and continuous integration.
35//!
36//! Testing with `gclient` is slower than `gtest` and produces more build
37//! artifacts, so it is better suited as the last mile in quality control.
38//! However, `gclient` gives the most accurate test results.
39//!
40//! # Usage
41//!
42//! To use the `gclient` library, you must import it into your `Cargo.toml` file
43//! in the `[dev-dependencies]` block. Also, you need to add some external
44//! crates that are used together with `gclient`:
45//!
46//! ```toml
47//! # ...
48//!
49//! [dev-dependencies]
50//! gclient = { git = "https://github.com/gear-tech/gear.git" }
51//! tokio = { version = "1.23.0", features = ["full"] }
52//!
53//! [patch.crates-io]
54//! sp-core = { git = "https://github.com/gear-tech/substrate.git", branch = "gear-stable" }
55//! sp-runtime = { git = "https://github.com/gear-tech/substrate.git", branch = "gear-stable" }
56//! ```
57//!
58//! Download the latest node binary for your operating system
59//! from <https://get.gear.rs>. Then unpack the package and run the node. Here
60//! we assume the node is running in developer mode:
61//!
62//! ```shell
63//! ./gear --dev
64//! ```
65//!
66//! The final step is to write tests in a separate `tests` directory and make
67//! `cargo` to execute them:
68//!
69//! ```shell
70//! cargo test
71//! ```
72//!
73//! # Examples
74//!
75//! Simple test example that uploads the program and sends the `PING` message.
76//!
77//! ```
78//! use gclient::{EventProcessor, GearApi, Result};
79//!
80//! const WASM_PATH: &str = "./target/wasm32-gear/release/first_gear_app.opt.wasm";
81//!
82//! #[tokio::test]
83//! async fn test_example() -> Result<()> {
84//!     // Create API instance
85//!     let api = GearApi::dev().await?;
86//!
87//!     // Subscribe to events
88//!     let mut listener = api.subscribe().await?;
89//!
90//!     // Check that blocks are still running
91//!     assert!(listener.blocks_running().await?);
92//!
93//!     // Calculate gas amount needed for initialization
94//!     let gas_info = api
95//!         .calculate_upload_gas(None, gclient::code_from_os(WASM_PATH)?, vec![], 0, true)
96//!         .await?;
97//!
98//!     // Upload and init the program
99//!     let (message_id, program_id, _hash) = api
100//!         .upload_program_bytes_by_path(
101//!             WASM_PATH,
102//!             gclient::now_micros().to_le_bytes(),
103//!             vec![],
104//!             gas_info.min_limit,
105//!             0,
106//!         )
107//!         .await?;
108//!
109//!     assert!(listener.message_processed(message_id).await?.succeed());
110//!
111//!     let payload = b"PING".to_vec();
112//!
113//!     // Calculate gas amount needed for handling the message
114//!     let gas_info = api
115//!         .calculate_handle_gas(None, program_id, payload.clone(), 0, true)
116//!         .await?;
117//!
118//!     // Send the PING message
119//!     let (message_id, _hash) = api
120//!         .send_message_bytes(program_id, payload, gas_info.min_limit, 0)
121//!         .await?;
122//!
123//!     assert!(listener.message_processed(message_id).await?.succeed());
124//!
125//!     Ok(())
126//! }
127//! ```
