19//! Functions and helpers for creating programs from programs.
20//!
21//! Any program being an actor, can not only process incoming messages and send
22//! outcoming messages to other actors but also create new actors. This feature
23//! can be useful when implementing the factory pattern, as a single
24//! actor can produce multiple derived actors with different input data.
25//!
26//! Firstly you need to upload a Wasm code of the future program(s) by calling
27//! `gear.uploadCode` extrinsic to obtain the corresponding
28//! [`CodeId`](crate::CodeId).
29//!
30//! You must also provide a unique byte sequence to create multiple program
31//! instances from the same code. This sequence is often referenced as _salt_.
32//! [`ProgramGenerator`] allows generating of salt automatically.
33//!
34//! The newly created program should be initialized using a corresponding
35//! payload; therefore, you must provide it when calling any `create_program_*`
36//! function.
